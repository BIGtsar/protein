## Выводы по графикам
Первый график:
Многопроцессорная обработка показывает наилучшую производительность при анализе больших объемов данных,обеспечивая ускорение до 3.5х по сравнению с однопоточной.
Многопоточность же ограничена GIL и уступает многопроцессорности,но при малых данных все же эффективна

Второй график:
На данном графике наблюдается аномальная ситуация — многопоточность показывает значительно лучшие результаты, чем многопроцессорность, что противоречит теории о преимуществе многопроцессорности для вычислительных задач в Python. 
Оптимум для обоих подходов достигается при 2 потоках/процессах, после чего производительность ухудшается.
